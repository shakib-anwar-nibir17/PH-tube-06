Discuss the scope of var, let, and const?

var, let, const are three most common used variable declaration method in JavaScript. All three of these three keywords have similarities in their syntax for declaring variables. Each three is unique on their own and has a variety of use cases. To understand the use cases of var, let and const, the following discussion will talk about the scope of each method.

Scope:
Before diving into the discussion for before mentioned methods, its important first have some about scope. Scope means the current context in which the code will be executed. Within a scope the values are visible and can be referenced within that scope. This allows the developer to manipulate the values inside the current scope. Before the introduction of ES6, JavaScript only had two types of scope-
1. Function Scope
2. Global Scope
After the introduction of ES6, as let and const keywords were also introduced, new scope named Block Scope stated to feature. A Block Scope is created with a pair of curly braces (“{ }”).

The scope of ‘var’:
var declarations are globally scoped or function scoped. The scope is global when var variable is declared outside of a function. When it is declared inside of the function it’s scope is limited to function scope. var can be redeclare within the same scope and will not result any error. var has very unique feature called hoisting which causes many bugs in the code. And that’s why let and const are now preferred declaration method.

The scope of ‘let’ and ‘const’:
Both declaration methods are block scope, meaning they can only accessed within their block. Unlike var, let can not redeclare but let does let you update its value. On the other hand const can neither be updated nor declared.

Tell us the use cases of null and undefined?

Use cases of ‘undefined’:
1. If a variable has been declared and a value has not been assigned to it , JavaScript automatically sets the value undefined. So undefined is used sometimes as variable initialization.
2. When functions are executed, if the function isn’t told explicitly what to return, it undefined as it’s value. This is most common in the case of most function that ends with return.
3.Trying to access a object property that doesn’t exist will result in undefined.

Use cases of ‘null’:
1. The term null is frequently used to express the lack of a value or to imply that a variable should not have a meaningful value.
2. You could set a variable to null in some instances to reset it or to signal that it no longer carries a valid value.
3.To indicate that a parameter is purposefully empty or has no value, functions may take null as an input.

What do you mean by REST API?

A set of limitations for using in the development of web services are defined by the architectural style known as Representational State Transfer (REST). REST API is a straightforward and flexible method of accessing web services without any processing. Because REST requires less bandwidth and is more appropriate for internet use, it is often preferred to the more reliable Simple Object Access Protocol (SOAP) technology. It is utilized to obtain or provide data from an online service. Through the REST API, only HTTP requests are used for communication. An API is a collection of specifications and protocols used to create and integrate application software. It is also referred to as a contract between an information provider and an information user, because it establishes the content necessary from the consumer (the call) and the material required by the producer (the answer). In other words, if you want to interact with a computer or system to get information or execute a function, an API lets you convey your desires to that system so that it can comprehend and fulfill your request. When a client requests a RESTful API, it sends a representation of the resource's state to the requester or endpoint. This data, or representation, is transmitted through HTTP in one of many formats: JSON (JavaScript Object Notation), HTML, XLT, Python, PHP, or plain text. JSON is the most widely used file format because, despite its name, it is language-independent and understandable by both people and machines.
